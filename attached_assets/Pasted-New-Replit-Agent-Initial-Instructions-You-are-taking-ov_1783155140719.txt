New Replit Agent — Initial Instructions

You are taking over an existing production project.

Phase 1 — Synchronize Only

Before doing anything else:

1. Synchronize the workspace with the latest GitHub repository.
2. Confirm:
    * Current branch
    * Current HEAD commit
    * Latest remote commit
    * Workspace is fully synchronized
3. Resolve any divergence safely.
4. Do not discard work.
5. Do not create new commits during this phase.
6. Ensure the workspace builds successfully.
7. Verify the application starts successfully.

When synchronization is complete, report:

* Branch
* Local commit SHA
* Remote commit SHA
* Workspace synchronized (Yes/No)

Stop if synchronization cannot be completed.

⸻

* Draft persistence

Understand the architecture before making changes.

⸻

Phase 4 — Fix Only These Issues

Fix only the following problems.

1. Property Location (Column C)

* Strip markdown formatting.
* Normalize input.
* Match against the Location LOV.
* Ignore formatting characters.
* Never fail because text contains bold, punctuation or spacing.

⸻

This is the highest priority issue.

Requirements:

* Never store a Google Maps URL.
* Detect society names reliably.
* Store gated community names.
* Store semi-gated apartment names when present.
* Leave blank for standalone properties.

⸻

3. Bathrooms and Balcony columns

Support common wording variations.

⸻

4. Tenant Type (Column M)
