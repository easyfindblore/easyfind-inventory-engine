## Routing

`webhookController.js` calls `inventoryController.tryHandleMessage()` first on every message. If it returns `true`, the legacy flow is skipped entirely. This is the single routing entry point — do NOT add a second one.

**Why:** The spec requires inventory sessions to intercept all messages (including plain text and media) during COLLECTING state without sending ACKs. A global first-pass check is the only way to maintain silence during collection.

**How to apply:** Any new broker-facing feature that needs to intercept messages before the legacy flow should extend `tryHandleMessage()`, not add its own check in webhookController.

## State machine

States: COLLECTING → INACTIVE → RETURN_MENU → COLLECTING (resume) or delete.
Also: CONFIRM_CANCEL, MID_SESSION_MENU, POST_SUCCESS, CONFIRM_DUPLICATE.

Sessions are keyed by `senderPhone`. One draft per broker at most.

## Durable persistence

`src/inventory/draftStore.js` — in-memory Map + Google Sheets "Drafts" tab (2-column: phone + JSON blob). Writes debounced 400ms. On startup, loads all rows back into memory. Auto-creates "Drafts" tab if missing.

**Why "Drafts" tab:** Google Sheets is the only database constraint. Filesystem on Render is ephemeral.

**Known limitation:** WhatsApp media IDs expire in 24 hours. Broker returning to an INACTIVE draft after >24h will get a "media expired" error at DONE time and must resend the photos.

## PID format

Old: `PIDyymmddNNN` (3-digit seq). New: `EF-YYYYMMDD-NNNNNN` (6-digit seq).
Both are generated by functions in `src/utils/pidGenerator.js`.
`generatePIDAndAppend` in `sheets.js` accepts `options.pidGenerator` — pass `generateEFPID` for new flow; defaults to `generatePID` for legacy.

## Triggers

New inventory triggers: "2", "Add Inventory", "Inventory", "Add" (whole message).
Legacy aliases also routed to new flow: "add property", "new property", "start property", "create listing".

## Required fields (DONE validation)

Location, Apartment Type, BHK, Bathrooms, Size, Furnishing, Tenant Type, Rent, Deposit, Availability. All 10 required. Missing fields message sent; session stays COLLECTING.

## Inactivity timer

60s poll. Nudge at 50% of timeout (default 15 min). State → INACTIVE at 100% (30 min). INACTIVE is preserved, never auto-deleted. Timer started by `inventoryController.initialize()` called from `src/index.js` startup.
