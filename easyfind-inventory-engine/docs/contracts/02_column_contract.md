# Purpose

This document defines how the Inventory Engine converts an incoming WhatsApp property message into values inside the Live Tracking sheet.

This document is ONLY responsible for parsing.

It does NOT define:

• Duplicate detection
• AI Mapping
• Business Rules
• Session Logic
• Google Sheet Operations

Those are separate documents.

---

# Principle

Every column in the inventory sheet has ONE destination.

The parser's only job is

Incoming Message

↓

Extract Value

↓

Assign to Correct Sheet Column

↓

Leave Blank if Missing

Nothing more.

No assumptions.

No business decisions.

---

# Parser Output

Parser always produces one Property Object.

Example

{
  "bhk":"2 BHK",
  "rent":"42000",
  "location":"Bellandur",
  "society":"Prestige Kew Gardens",
  ...
}

This object is later validated before inserting into Google Sheets.

# Multiple Messages

Example

Photo

---

# Multiple Values

If duplicate field appears

Latest value wins.

Example

Rent: 40000

Later

Rent: 42000

↓

Stored

42000

---

# Unknown Fields

If parser finds unknown information

Ignore.

Do not create custom columns.

---

# Failure

Parser never throws an error for missing values.

Missing values remain blank.

---

# Parser Responsibilities

✔ Read merged session

✔ Extract known fields

✔ Produce Property Object

✔ Preserve raw values

✔ Leave unknown values blank

✔ Never map values

✔ Never detect duplicates

✔ Never upload media

✔ Never write to Google Sheets

Those responsibilities belong to later modules.
